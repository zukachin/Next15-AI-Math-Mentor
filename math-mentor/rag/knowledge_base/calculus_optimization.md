# Calculus – Optimization (JEE-style)

## First derivative test (critical points)
- Find f'(x); set f'(x) = 0 (and check points where f' does not exist). These are critical points.
- Sign of f': f' > 0 ⇒ f increasing; f' < 0 ⇒ f decreasing. Local maximum where f' changes from + to -; local minimum where f' changes from - to +.

## Second derivative test
- At a critical point c with f'(c) = 0: if f''(c) > 0 then local minimum; if f''(c) < 0 then local maximum; if f''(c) = 0 then test is inconclusive (use first derivative test or higher derivatives).

## Closed interval method (absolute max/min on [a, b])
1. Find critical points of f in (a, b) (where f'(x) = 0 or f' undefined).
2. Evaluate f at all critical points and at endpoints a and b.
3. Largest value = absolute maximum; smallest = absolute minimum.

## Domain and constraints
- If the problem restricts x (e.g. x > 0, or x in [0, 10]), only consider critical points inside that domain and include boundary points in the list to evaluate.
- For “minimum cost” or “maximum area” word problems, write the quantity to optimize as a function of one variable using the constraint, then apply the above.

# Optimization – Common mistakes and pitfalls

- Forgetting to check endpoints when the domain is a closed interval; absolute extrema can occur at boundaries.
- Assuming every critical point is a max or min; verify with first or second derivative test.
- Wrong second derivative test: f''(c) > 0 gives minimum, f''(c) < 0 gives maximum (not the other way around).
- Not stating the full answer: give both the point (x value) and the function value f(x).
- In word problems, not using the constraint to reduce to one variable before differentiating.
